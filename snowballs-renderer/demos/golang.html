Sure, here’s a JavaScript function that can render different types of content. For simplicity, let’s create a function that can render either plain text, HTML, or an image.

Step-by-Step Explanation

  1. Function name: renderContent

  2. Parameters:

    • type: The type of content to render ('text', 'html', or 'image').

    • content: The content to render.

    • targetElementId: The ID of the HTML element where the content should be rendered.

JavaScript Code

function renderContent(type, content, targetElementId) {
        // Get the target element by its ID
        const targetElement = document.getElementById(targetElementId);
    
        // Ensure the target element exists
        if (!targetElement) {
            console.error(`Element with ID '${targetElementId}' not found.`);
            return;
        }
    
        // Render content based on the type
        switch (type) {
            case 'text':
                // Render plain text
                targetElement.textContent = content;
                break;
            case 'html':
                // Render raw HTML (use with caution)
                targetElement.innerHTML = content;
                break;
            case 'image':
                // Create an image element and set its source
                const img = document.createElement('img');
                img.src = content;
                // Append the image to the target element
                targetElement.appendChild(img);
                break;
            default:
                console.error(`Unsupported content type '${type}'. Use 'text', 'html', or 'image'.`);
                break;
        }
    }

Example Usage

  1. HTML structure:

<div id="contentArea"></div>

  1. JavaScript usage:

// Render plain text
    renderContent('text', 'Hello, world!', 'contentArea');
    
    // Render HTML
    renderContent('html', '<strong>Bold Text</strong>', 'contentArea');
    
    // Render an image
    renderContent('image', 'https://via.placeholder.com/150', 'contentArea');

Additional Notes

  • Security: Be cautious when using innerHTML, as it can make your application vulnerable to Cross-Site Scripting (XSS) attacks.
